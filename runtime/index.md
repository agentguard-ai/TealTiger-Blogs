# Runtime

Runtime governance focuses on what **actually happens at execution time**, not what was intended at design time.

This category covers how enforcement, observability, and detection operate while AI agents are running — across tools, APIs, memory, and external systems.

At runtime, policy decisions must be **deterministic, inspectable, and auditable**.

## What You’ll Find Here

- Runtime security signals
- Execution‑time policy enforcement
- Observable agent behavior
- Deterministic governance primitives

Content in this category evolves as new runtime mechanisms are introduced.
